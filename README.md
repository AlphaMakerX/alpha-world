# Alpha World（给大模型开发用）

## 1. 游戏定位
- 多人在线地图经营游戏（地产经营 + 生产销售）。
- 多个玩家在同一张地图中实时互动，核心是资源管理、地块竞争、经济循环。

## 2. 核心规则
- 地图包含多个地块（Tile）。
- 玩家有体力槽（Stamina），每次掷骰子会消耗体力。
- 体力为 0 时无法继续掷骰子。
- 玩家移动到可购买地块时可进行购买。
- 已购买地块可建设建筑：住宅、商店、工厂。
- 其他玩家停留在你的地块时，需要支付费用。
- 工厂可加工商品，商店可上架并售卖商品。

## 3. 核心玩法循环（Gameplay Loop）
1. 玩家消耗体力掷骰子。
2. 按点数移动到目标地块。
3. 执行地块交互（购买、缴费或其他事件）。
4. 在自有地块上建造/升级建筑。
5. 工厂生产商品，商店售卖商品，获得金币收益。
6. 进入下一轮，持续经营和竞争。

## 4. 系统模块建议（供大模型生成代码）
- 玩家系统：登录、在线状态、体力恢复、资产管理。
- 地图系统：地块配置、地块所有权、地块事件触发。
- 行动系统：掷骰子、移动、购买、建造、收费。
- 建筑系统：住宅/商店/工厂的建造、升级、收益逻辑。
- 商品系统：商品定义、加工配方、库存管理。
- 经济系统：金币收支、租金结算、商品售卖结算。
- 实时同步：WebSocket 广播玩家移动、资产变化、事件结果。

## 5. 核心数据模型（建议）
- `Player`: id, nickname, stamina, coins, position, inventory
- `Tile`: id, type, ownerId, price, toll, buildingType, level
- `Building`: id, tileId, type, level, config
- `Item`: id, name, basePrice, recipe
- `MatchState`: id, mapSeed, players, turnIndex, phase
- `TransactionLog`: id, fromPlayerId, toPlayerId, amount, reason, timestamp

## 6. MVP 最小可用范围
- 单地图房间（2~4 人）。
- 掷骰子移动 + 体力消耗/限制。
- 地块购买 + 基础收费。
- 三类建筑可建造（先不做复杂升级树）。
- 工厂产出 1~2 种基础商品，商店可售卖并获得金币。
- 实时广播关键事件（移动、购买、收费、售卖）。
