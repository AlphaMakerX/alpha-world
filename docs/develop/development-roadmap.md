# Alpha World 开发路线图

> 最后更新: 2026-04-11

---

## 项目现状总结

Alpha World 是一款基于浏览器的 2D 多人经济模拟游戏，采用 Next.js 16 + Phaser 3 + tRPC + Drizzle ORM 技术栈，遵循 DDD 架构。

**已完成的核心系统：**
- 用户注册/登录（NextAuth）
- 2D 世界地图（Phaser 引擎，键盘移动）
- 地块购买系统
- 建筑系统（住宅/工厂/商店/收购站）
- 工厂生产系统（采购/加工/组装，21 种物品）
- 商店交易系统（P2P）
- 收购站系统（NPC 买单）
- 背包/库存系统
- Adam 央行经济系统
- 体力系统（移动不消耗，自然恢复）

---

## 阶段一：Bug 修复与稳定性（优先级：高）

### 1.1 位置同步 Bug
- **问题：** 玩家坐标 2 秒同步一次，导致刷新页面或建造后角色回到旧坐标
- **方案：** 在关键操作（建造、刷新）前立即同步坐标；页面关闭前（beforeunload）发送最终坐标
- **涉及：** `world-map` 相关 hooks、`updateUserPosition` mutation

### 1.2 页面状态刷新
- **问题：** 金钱和背包物品在操作后不能及时刷新
- **方案：** 在 mutation 的 `onSuccess` 中精确 invalidate 相关 query（money、inventory）
- **涉及：** 各 feature 的 mutation hooks

---

## 阶段二：核心玩法增强（优先级：高）

### 2.1 体力系统完善
- **目标：** 让体力成为有意义的游戏资源
- **方案：**
  - 工厂生产消耗体力（按制造时长比例）
  - 体力上限 1000，自然缓慢恢复
  - 住宅建筑提供加速恢复：消耗时间 + 金钱，或消耗特定物品
- **涉及：** `player-stamina.ts`、factory use case、新增住宅恢复 use case

### 2.2 住宅建筑功能
- **目标：** 让住宅建筑有实际用途
- **方案：**
  - 进入自己的住宅可以「休息」，加速恢复体力
  - 休息消耗金钱或物品（如食物类物品）
  - 可考虑增加「食物」类配方（面包、饮料等）
- **新增：** 住宅交互 UI、休息 use case、食物配方

### 2.3 邮箱系统
- **目标：** 统一物品/金钱流转的中转机制，解决页面刷新问题
- **方案：**
  - 新增 `mailbox` 表，存储待领取的物品和金钱
  - 工厂产出 → 邮箱 → 玩家手动领取
  - 商店交易 → 邮寄金钱给卖家 + 邮寄物品给买家
  - 收购站 → 邮寄金钱给卖家 + 邮寄物品给站主
  - 支持玩家间直接邮寄物品/金钱
- **新增：** mailbox bounded context（schema、repository、use case、router、UI）

## 阶段三：游戏体验优化（优先级：中）

### 3.1 世界地图组件重构
- **问题：** `world-map` 组件过于臃肿，顶部功能区承载太多
- **方案：**
  - 拆分为独立子组件：PlayerHUD、MiniMap、ActionBar、PlotInfoPanel
  - 抽取自定义 hooks：usePlayerMovement、usePlotInteraction、useGameCamera
  - Phaser Scene 与 React UI 解耦

### 3.2 UI 体验提升
- **背包 UI 优化：** 网格布局、物品图标、数量显示、分类筛选
- **配方/制造模块优化：** 更清晰的配方树展示、材料需求高亮、一键制造
- **地块信息优化：** 悬浮预览、所有者信息、建筑状态一目了然
- **移动端适配：** 虚拟摇杆、触屏交互

### 3.3 Loading 与错误处理
- **方案：** 全局 loading skeleton、操作中按钮禁用、tRPC error boundary 统一处理
- **涉及：** 各 feature 组件、app-providers

---

## 阶段四：社交与互动（优先级：中）

### 4.1 多人实时交互
- **目标：** 能看到其他玩家在地图上移动
- **方案：** WebSocket / SSE 推送其他玩家位置
- **技术：** 可考虑 tRPC subscription 或独立 WebSocket 服务

### 4.2 聊天系统
- **目标：** 玩家间基础交流
- **方案：** 全局聊天频道 + 私聊
- **新增：** chat bounded context

### 4.3 排行榜与成就
- **目标：** 增加竞争性和目标感
- **方案：**
  - 扩展现有财富排行榜
  - 新增：生产量排行、交易量排行、地块数量排行
  - 成就系统：首次建造、首次交易、收集全部物品等

---

## 阶段五：经济系统深化（优先级：中低）

### 5.1 市场价格机制
- **目标：** 动态供需影响价格
- **方案：**
  - 物品历史价格记录与趋势展示
  - Adam 的收购/出售价格根据流通量动态调整
  - 市场行情面板

### 5.2 税收与费率系统
- **目标：** 经济调控工具
- **方案：**
  - 交易税（P2P 交易抽成）
  - 地块持有税（定期扣除，鼓励活跃使用）
  - 税率可由系统动态调整

### 5.3 拍卖行
- **目标：** 更高效的交易方式
- **方案：** 全局拍卖行（不依赖商店建筑），支持竞价、一口价

---

## 阶段六：基础设施与部署（优先级：高，可并行）

### 6.1 生产环境部署
- **目标：** 上线可访问的版本
- **方案：**
  - 部署平台选择（Vercel / 自建服务器）
  - PostgreSQL 云数据库（Supabase / Neon / RDS）
  - 环境变量管理、CI/CD pipeline
  - 域名与 HTTPS

### 6.2 测试覆盖
- **目标：** 保障核心逻辑正确性
- **方案：**
  - 补充 domain 层单元测试（实体、值对象）
  - use case 集成测试（含数据库）
  - tRPC router E2E 测试
  - 利用现有 Vitest 配置

### 6.3 性能优化
- **数据库：** 关键查询加索引、连接池配置
- **前端：** Phaser 渲染优化（大量地块时）、React Query 缓存策略
- **API：** 批量查询优化、分页
