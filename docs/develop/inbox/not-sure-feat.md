# 待评审 / 不确定

以下方向尚未拍板，先集中记在此文件，不写入 [PLAN.md](../PLAN.md)。

## 1. 体力管理

- 体力上限 1000，每分钟恢复 1 点；在用户下次获取体力值时结算恢复。
- 制造配方消耗体力，数值与时间成正比。
- 住宅可生产体力。

## 2. 住宅管理

- 老板：设定价格，由系统按固定税率扣除。
- 旅客：类似生产流程，输入数量，开启体力生产，消耗金钱。

## 3. 邮箱管理

- 可给他人邮寄物品、金钱。
- 生产产出先进入邮箱，点击后再入背包。
- 商店购买后：邮寄金钱给店主；邮寄物品给买家。
- 收购站提交后：邮寄金钱给用户；邮寄物品给收购站主人。

### 邮箱与建筑（疑问）

- 邮箱作为「建筑功能」是否有必要：若目的仅为让页面正确刷新（金钱、背包物品），定时刷新与手动刷新当前也能成功，是否仍需要单独邮箱建筑与流程。

## 4. 制造稀缺性，迫使玩家协作

当前每个玩家可独立完成全部生产链，缺乏交易动机。需要从机制上制造"我有你没有"的依赖关系。

- **资源区域化**：不同区域的地块只能采集特定原材料（比如 P1-P2 行产木材，P5-P6 行产铁矿），玩家必须交易才能获得全品类原料。
- **工厂配方解锁机制**：工厂需要购买配方才能生产对应物品；工厂分不同类型（如冶炼厂、纺织厂、组装厂），各自拥有不同的配方池；工厂需要升级才能解锁更高级的配方。玩家无法用单一工厂覆盖所有产线，进一步强化专业化分工和交易需求。
