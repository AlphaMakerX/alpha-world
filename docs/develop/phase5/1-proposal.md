# Phase 5 业务规则：体力系统

## 一、体力属性

| 属性 | 值 |
|------|-----|
| 体力上限 | 1000 |
| 新玩家初始体力 | 1000（满） |
| 自然恢复速率 | 10 点/小时（≈0.00278/秒） |
| 满恢复时间 | ≈100 小时（0→1000） |

自然恢复持续进行，无需任何操作，体力不超过上限。

## 二、体力消耗

唯一消耗场景：**工厂生产**。

### 消耗公式

```
体力消耗 = 配方时长(秒) × 生产数量 × 0.1
```

### 消耗示例（×1 数量）

| 配方类型 | 时长范围 | 体力消耗 |
|---------|---------|---------|
| 采购 procurement | 5–10s | 0.5–1 |
| 加工 processing | 25–80s | 2.5–8 |
| 组装 assembly | 75–600s | 7.5–60 |

设计意图：
- 采购几乎不消耗体力，入门友好
- 加工中等消耗，可连续做 125–400 次
- 高级组装消耗大，需规划体力分配
- 终极配方（600s）单次消耗 60，占上限 6%

### 体力不足规则

- 启动生产前先结算自然恢复，再判断体力是否充足
- 体力不足时**拒绝启动生产**，不允许透支

## 三、住宅休息

住宅建筑新增「休息」功能，花费金币 + 等待时间，恢复体力。

### 休息参数

| 属性 | 值 |
|------|-----|
| 档位 | 单一档位 |
| 等待时间 | 300 秒（5 分钟） |
| 恢复量 | 1000（恢复满，超过上限截断） |
| 收取方式 | 手动收取（到时间后需主动领取） |
| 并发限制 | 每栋住宅同时只能有 1 个休息任务 |

对比自然恢复（10/小时，免费）：休息 5 分钟恢复满，约 1200 倍速，但花钱。适合体力见底又急需生产的场景。

### 在自己的住宅休息

- 费用：**500 金币**，支付给系统（Adam）
- 前提：玩家是该住宅地块的所有者

### 在别人的住宅休息

- 费用：由**住宅主人自由定价**（无上下限）
- 系统抽成：总费用的 **10%** 归系统（Adam）
- 主人收入：总费用的 **90%** 归住宅主人
- 前提：住宅主人已设定价格

示例：主人定价 1000 金币 → 玩家支付 1000，其中 100 归 Adam，900 归主人。

### 休息流程

1. 玩家对住宅发起「休息」
2. 校验金币充足 + 该住宅无进行中的休息任务
3. 扣除金币（按自己/别人住宅分别处理收款方）
4. 开始休息，等待 300 秒
5. 到时间后，玩家手动「收取」
6. 收取时先结算自然恢复，再加上休息恢复量（截断到上限）

## 四、约束与边界

- 不改变现有配方内容和生产时长
- 不改变其他建筑类型的行为
- 体力仅影响工厂生产，不影响建造、交易等其他操作
- 住宅休息恢复量固定为 1000，不因当前体力而变化（超出上限截断）
