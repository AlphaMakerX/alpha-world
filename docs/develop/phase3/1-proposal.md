# Phase 3 Proposal：制造稀缺性 — 工厂配方解锁与专业化分工

## 目标

通过**工厂类型化、配方付费解锁、工厂升级**三项机制，打破当前「任一玩家用一座工厂即可生产所有物品」的自给自足局面，制造结构性的资源不对称，迫使玩家之间产生真实的交易动机与协作关系。

## 现状与问题

### 当前机制

- 工厂只有一种类型（`factory`），所有工厂共享完整的 54 条配方池（采购 13 / 加工 24 / 组装 17）。
- 任何工厂建成即可使用全部配方，无需额外解锁。
- 工厂没有等级概念（数据库有预留字段但未使用）。
- 玩家只要拥有一块地 + 一座工厂 + 足够金币，就能独立完成从原材料到高级商品的全部生产链。

### 核心问题

- **交易动机为零**：玩家没有理由向其他玩家购买物品，因为自己能生产一切。
- **商店和收购站形同虚设**：这两种建筑类型缺乏使用场景。
- **玩家之间无依赖关系**：无法形成经济网络，世界缺乏社交驱动力。

## 业务规则

### 一、工厂类型

#### 1.1 类型定义

工厂建造时必须选择一种**工厂类型**（factory subtype），不同类型拥有不同的**可用配方池**。类型一经选定，不可更改。

引入以下工厂类型（具体类型与配方的映射关系在 design 中确定）：

| 工厂类型 | 定位 | 配方池方向 |
|---------|------|-----------|
| 采矿场 (mine) | 原材料采集 | 矿石、煤炭、石材、沙子、粘土等基础资源的采购配方 |
| 伐木场 (lumber_mill) | 原材料采集 | 木材等基础资源的采购配方 |
| 纺织厂 (textile_mill) | 原材料采集 + 加工 | 棉花、亚麻采购，布料、亚麻布加工 |
| 牧场 (ranch) | 原材料采集 + 加工 | 生皮、动物脂肪采购，皮革、牛油、蜡烛加工 |
| 炼金坊 (apothecary) | 原材料采集 + 加工 | 草药采购，染料、墨水、药品加工 |
| 造水厂 (waterworks) | 原材料采集 | 高效产水的独占配方（普通产水配方所有工厂共享） |
| 冶炼厂 (smelter) | 加工 | 铁锭、钢铁等金属加工配方 |
| 木工坊 (carpentry) | 加工 + 组装 | 木板、家具等木材加工与组装配方 |
| 造纸厂 (paper_mill) | 加工 + 组装 | 纸张、书籍等配方 |
| 组装厂 (assembler) | 高级组装 | 工具、机械零件、雕塑等多材料组装配方 |

- 每种工厂类型的配方池**互有重叠但不完全相同**——某些基础配方可能多个类型共享，高级配方则倾向独占。
- **水（water）的采购配方为通用配方**，所有工厂类型均可使用；造水厂拥有独占的高效产水配方（同等金币产出更多水或生产时间更短）。
- 具体的「类型 → 可用配方列表」映射表在 design 阶段定稿，proposal 只确定分类原则。

#### 1.2 建造变更

- 建造工厂时，玩家需要额外指定 `factorySubtype` 参数。
- 不同工厂类型的建造费用**可以不同**（具体数值 design 决定）。
- 建造接口校验：`factorySubtype` 必须是有效的工厂类型枚举值，否则拒绝。

#### 1.3 对现有工厂的兼容

- 当前处于开发阶段，采用**清档策略**：上线时清除所有已有工厂数据，玩家重新建造并选择工厂类型。

### 二、配方解锁

#### 2.1 解锁机制

- 工厂建成后**不自带任何配方**（或仅自带该类型的最基础配方，design 决定）。
- 玩家需为每座工厂**单独购买**配方，花费金币将配方解锁到该工厂上。
- 配方解锁是**工厂级别**的：同一玩家的两座同类型工厂，各自独立解锁，互不共享。
- 已解锁的配方**永久有效**，不会过期或被回收。

#### 2.2 解锁条件

每个配方有以下解锁前置条件：

- **工厂类型匹配**：该配方必须属于该工厂类型的可用配方池。
- **工厂等级达标**：该配方要求的最低工厂等级 ≤ 工厂当前等级（见下节）。
- **金币充足**：玩家持有的金币 ≥ 配方的解锁费用。

三项条件全部满足时，扣除金币，配方立即可用。

#### 2.3 解锁费用

- 每个配方有一个固定的解锁费用（金币数额），由配方定义决定。
- 高级配方的解锁费用应高于基础配方（具体数值 design 决定）。
- 费用扣除后转入系统账户（Adam），与当前生产扣费逻辑一致。

#### 2.4 对生产流程的影响

- 启动生产时，额外校验：**该工厂是否已解锁所选配方**。
- 未解锁的配方不允许启动生产，返回明确的错误信息（如「该工厂尚未解锁此配方」）。
- 查询工厂可用配方时，应区分「该类型可用但未解锁」和「已解锁」两种状态，便于玩家决策。

### 三、工厂等级

#### 3.1 等级定义

- 每座工厂拥有一个**等级**（level），初始为 1 级。
- 等级上限由 design 决定（建议 3–5 级）。
- 等级影响：**可解锁的配方范围**——更高级的配方要求更高的工厂等级。

#### 3.2 升级机制

- 玩家可花费金币（和/或物品）对工厂进行升级。
- 升级为**即时生效**，不需要等待时间（是否引入升级等待时间 design 决定）。
- 每级升级费用递增（具体数值 design 决定）。
- 升级费用扣除后转入系统账户（Adam）。

#### 3.3 升级条件

- 工厂当前等级 < 等级上限。
- 玩家金币（和/或物品）充足。
- 工厂当前没有进行中的生产任务（是否要求这一点 design 决定）。

### 四、稀缺性设计原则

以下原则指导 design 阶段的具体数值与映射配置：

#### 4.1 不可能独占

- **没有任何单一工厂类型能覆盖全部配方**——玩家想要获取所有种类的物品，必须建造多种类型的工厂，或向其他玩家交易。
- 考虑到地块数量有限（每块地只能建一座建筑），玩家很难同时拥有覆盖全产线的工厂集合。

#### 4.2 依赖链

- 高级组装配方的输入材料，应来自**不同工厂类型**的产出。
- 例如：组装家具需要木板（木工坊产出）+ 布料（纺织厂产出）+ 铁钉（冶炼厂产出）。
- 单一玩家除非拥有多座不同类型的工厂，否则必须从他人处购买中间材料。

#### 4.3 梯度稀缺

- 基础材料（base_material 层级）：获取门槛低，多数玩家都能生产，供给充足。
- 加工品（processed_goods 层级）：需要特定类型工厂 + 一定等级，供给开始收窄。
- 高级商品（advanced_goods 层级）：需要高等级工厂 + 高价解锁费 + 多种跨类型输入，能生产的玩家少，交易价值高。

#### 4.4 入门友好

- 新玩家仍可通过建造一座工厂 + 解锁基础配方快速开始游戏。
- 稀缺性的压力主要出现在**中后期**——当玩家想生产高级商品时，才会真正感受到专业化分工的必要。

## 约束与排除范围

- **不改变现有物品体系**：不新增物品、不修改物品分层（base_material / processed_goods / advanced_goods），仅在工厂侧加约束。
- **不改变配方内容**：现有 21 条配方的输入、输出、时长保持不变，仅增加「归属哪些工厂类型」和「解锁费用 / 等级要求」等元数据。
- **不改变商店和收购站逻辑**：交易机制保持现状，本阶段只制造交易动机。
- **不引入自动定价 / 拍卖行**：价格仍由玩家自行决定。
- **不引入工会 / 联盟系统**：协作通过自然的交易行为达成，不引入社交组织结构。
- **不包含前端 UI 实现**：本 proposal 只讨论规则，UI 呈现方式属于 design 范畴。
- **不包含数值平衡调优**：具体费用、等级上限、配方映射等数值在 design 阶段确定，上线后可通过配置调整。

## 验收（产品视角）

- **工厂类型化**：建造工厂时必须选择类型；不同类型的工厂在查询配方时返回不同的可用配方列表。
- **配方需解锁**：新建工厂无法直接使用配方生产；玩家需花费金币为工厂解锁配方后才能启动生产。
- **工厂可升级**：玩家可花费金币升级工厂等级；升级后可解锁更高级的配方。
- **稀缺性生效**：单一玩家使用单一工厂，无法生产全部物品；要获取全部种类物品，必须建造多种类型的工厂，或通过商店 / 收购站与其他玩家交易。
- **新玩家不受阻**：新玩家建造一座工厂后，能以合理的初始金币解锁至少 1–2 个基础配方并完成一次生产。
- **清档上线**：开发阶段采用清档策略，无需向下兼容。
